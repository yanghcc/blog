# eslint规范

TIP

= eslint规范 =

官方地址：http://eslint.org/

EsLint帮助我们检查Javascript编程时的语法错误。比如：在Javascript应用中，你很难找到你漏泄的变量或者方法。EsLint能够帮助我们分析JS代码，找到bug并确保一定程度的JS语法书写的正确性。

EsLint是建立在Esprima(ECMAScript解析架构)的基础上的。Esprima支持ES5.1,本身也是用ECMAScript编写的，用于多用途分析。EsLint不但提供一些默认的规则（可扩展），也提供用户自定义规则来约束我们写的Javascript代码。

# 规范说明

EsLint提供以下支持：

  1. ES6
  2. AngularJS
  3. JSX
  4. Style检查
  5. 自定义错误和提示

EsLint提供以下几种校验：

  1. 语法错误校验
  2. 不重要或丢失的标点符号，如分号
  3. 没法运行到的代码块（使用过WebStorm的童鞋应该了解）
  4. 未被使用的参数提醒
  5. 漏掉的结束符，如 }
  6. 检查变量的命名

规则的错误等级有三种：

  • 0：关闭规则。
  • 1：打开规则，并且作为一个警告（不影响exit code）。
  • 2：打开规则，并且作为一个错误（exit code将会是1）。

== angular == angular 代码风格由eslint-plugin-angular提供

# 可能的错误

以下内容会导致错误

  • module-getter - 禁止引用以变量形式引用模块 使用angular.module('myModule')代替 (y022)
  • module-setter - 禁止使用变量形式定义模块(linked to module-getter (y021)
  • no-private-call - 禁止使用angular内部以$$开头的方法

# 最佳实践

下面的规则可以避免一些问题

  • component-limit - 限制单个文件中 components 的数量(y001)
  • controller-as-route - 在 routes 或 states中 必须使用controllerAs (y031)
  • controller-as-vm - 在controller中，不直接使用this，而是使用一个特殊的变量如 vm代替它 (y032)
  • controller-as - 在controller中禁止分配任务给 $scope，$scope只作为watchers的绑定对象，值绑定在this上  (y031)
  • deferred - 使用 $q(function(resolve, reject){}) 代替 $q.deferred
  • di-unused - 不允许未使用的DI引用参数
  • directive-restrict - 限制directive的restrict的类型， 默认只能是AE (y074)
  • empty-controller - 是否限制controller的内容不能为空
  • no-controller - 避免使用 controller(由compoent优先)
  • no-inline-template - 禁止单行的template ，而是以路径引入
  • no-run-logic - 保持 run 函数的整洁，不在run中执行函数 (y171)
  • no-services - disallow DI of specified services for other angular components ($http for controllers, filters and directives)
  • on-watch - $on和$watch 必须保存在一个变量下，用于$destroy
  • prefer-component -

# 禁用angular的某些功能

  • no-cookiestore - 使用 $cookies 代替 $cookieStore
  • no-directive-replace - 禁止directive 指令的replace属性
  • no-http-callback - 禁止  $http的 success() 和 error()方法

# 命名

These rules help you to specify several naming conventions.

  • component-name - 为所有的component设置统一的前缀
  • controller-name - 为所有的controller设置统一的前缀 (y123, y124)
  • directive-name -为所有的directive设置统一的前缀 (y073, y126)
  • file-name - require and specify a consistent component name pattern (y120, y121)
  • filter-name - 为所有的filter块设置统一的前缀
  • module-name - 为所有的module设置统一的前缀 (y127)
  • service-name - 为所有的service设置统一的前缀 (y125)

# Conventions

Angular often provide multi ways to to something. These rules help you to define convention for your project.

  • di-order -依赖的 DI（注入的服务）参数必须以字母顺序排列
  • di - require a consistent DI syntax
  • dumb-inject - unittest inject functions should only consist of assignments from injected values to describe block variables
  • function-type - require and specify a consistent function style for components ('named' or 'anonymous') (y024)
  • module-dependency-order - require a consistent order of module dependencies
  • no-service-method - use factory() 代替 service() (y040)
  • one-dependency-per-line - require all DI parameters to be located in their own line
  • rest-service - disallow different rest service and specify one of '$http', '$resource', 'Restangular'
  • watchers-execution - require and specify consistent use $scope.digest() or $scope.apply()

# 使用angular内部的方法

下部的规则可以强制使用angular内部的定义的方法

  • angularelement - 使用angular.element 代替 $ 和 jQuery
  • definedundefined - 使用 angular.isDefined和angular.isUndefined 代替 other undefined checks
  • document-service - 使用 $document 代替 document (y180)
  • foreach - 使用 angular.forEach 代替 native Array.prototype.forEach
  • interval-service - 使用 $interval 代替 setInterval (y181)
  • json-functions - 使用 angular.fromJson and 'angular.toJson' 代替 JSON.parse和JSON.stringify
  • log - 使用 the $log service 代替 the console methods
  • no-angular-mock - require to 使用 angular.mock methods directly
  • no-jquery-angularelement - disallow to wrap angular.element objects with jQuery or $
  • timeout-service - 使用 $timeout 代替 setTimeout (y181)
  • typecheck-array - 使用 angular.isArray 代替 typeof comparisons
  • typecheck-date - 使用 angular.isDate 代替 typeof comparisons
  • typecheck-function - 使用 angular.isFunction 代替 typeof comparisons
  • typecheck-number - 使用 angular.isNumber 代替 typeof comparisons
  • typecheck-object - 使用 angular.isObject 代替 typeof comparisons
  • typecheck-string - 使用 angular.isString 代替 typeof comparisons
  • window-service - use $window 代替 window (y180)
